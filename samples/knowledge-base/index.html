Welcome to the Engineering Team

Welcome to the Engineering organization at Contoso. This onboarding guide is designed to help you get up and running quickly. Whether you are joining as a junior developer or a seasoned architect, this document covers everything you need to know during your first two weeks, from setting up your development environment to understanding our deployment workflows.

Our engineering culture values collaboration, quality, and continuous improvement. We believe in writing clean, well-tested code and sharing knowledge openly across the team. This wiki is one of the main ways we preserve and distribute institutional knowledge.

Development Environment Setup

Once you have the required tools installed, follow these steps to set up the main application repository. Our monorepo uses Turborepo for build orchestration and pnpm as the package manager.

  1. Clone the main repository from Azure DevOps using the SSH URL provided in your onboarding email.
  2. Run pnpm install at the root of the repository to install all dependencies.
  3. Copy .env.example to .env.local and fill in the values from the team's shared vault.
  4. Run pnpm dev to start the development server on localhost:3000.
  5. Verify everything works by running pnpm test to execute the unit test suite.

Our applications follow a microservices architecture pattern. Each service communicates through our internal API gateway. We deploy to AKS clusters using a canary deployment strategy. Feature flags are managed through our feature management platform, which enables safe experimentation and gradual rollouts to production users.

Code Review Process

All changes go through a pull request workflow in Azure DevOps. Every PR requires at least two approvals from team members, one of whom must be a designated code owner for the modified paths. Automated checks include linting, unit tests, integration tests, and a build verification step. We aim for a review turnaround of less than 24 hours for standard PRs.

On-Call Rotation

After your first month, you will be added to the on-call rotation. On-call engineers are responsible for triaging incoming alerts, coordinating incident response, and writing post-incident reviews. Detailed runbooks for common scenarios are available in the Operations section of this knowledge base.
